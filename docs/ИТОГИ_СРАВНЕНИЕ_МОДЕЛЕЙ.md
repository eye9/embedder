## ⚠️ Важные моменты

### Размерность эмбеддингов

Разные модели создают эмбеддинги **разной размерности**:

| Модель | Размерность |
|--------|-------------|
| FRIDA | 1536 |
| E5-Small | 384 |
| MiniLM-L12 | 384 |
| RuBERT-Tiny2 | 312 |

**Это означает:**
- ❌ Нельзя использовать базу, созданную с FRIDA, для E5-Small
- ✅ Нужно пересоздавать базу при смене модели
- ⚠️ `compare_models.py` покажет поиск только для текущей модели базы

### Что работает без пересоздания базы

✅ **Можно тестировать:**
- Скорость загрузки модели
- Скорость генерации эмбеддингов
- Размер модели в памяти

❌ **Требует пересоздания базы:**
- Тестирование поиска
- Сравнение качества результатов
- Сравнение similarity scores

### Рекомендуемый workflow

```bash
# Шаг 1: Быстрое сравнение всех моделей (без поиска)
python compare_models.py

# Шаг 2: Выберите понравившуюся модель

# Шаг 3: Протестируйте её с поиском
python test_single_model.py intfloat/multilingual-e5-small cuda

# Шаг 4: Если база несовместима, пересоздайте
# Обновите config.yaml
python load_tnved.py

# Шаг 5: Повторите тест
python test_single_model.py intfloat/multilingual-e5-small cuda
```

## 💡 Рекомендации

### Для GTX 1060 3GB + CUDA
```yaml
model:
  name: "intfloat/multilingual-e5-small"
  device: "cuda"
processing:
  batch_size: 32
```

**Преимущества:**
- ✅ В 6.5 раз меньше памяти
- ✅ В 3-5 раз быстрее
- ✅ Отличное качество
- ✅ Стабильная работа

### Для CPU-only систем
```yaml
model:
  name: "cointegrated/rubert-tiny2"
  device: "cpu"
processing:
  batch_size: 100
```

**Преимущества:**
- ✅ Самая быстрая на CPU
- ✅ Минимальные требования
- ✅ Специально для русского

### Для максимального качества
```yaml
model:
  name: "ai-forever/FRIDA"
  device: "cpu"
processing:
  batch_size: 50
```

**Когда использовать:**
- Качество важнее скорости
- Достаточно мощное железо
- Офлайн обработка

## 🔧 Устранение проблем

### Ошибка: "Collection expecting embedding with dimension of X, got Y"

**Причина:** База создана с другой моделью

**Решение:**
```bash
# Обновите config.yaml с нужной моделью
python load_tnved.py
```

### Ошибка: "CUDA out of memory"

**Решение:**
1. Уменьшите `batch_size` в config.yaml
2. Используйте более компактную модель
3. Переключитесь на CPU

### compare_models.py показывает "N/A*"

**Это нормально!** База создана с другой моделью. Используйте `test_single_model.py` для тестирования конкретной модели.
