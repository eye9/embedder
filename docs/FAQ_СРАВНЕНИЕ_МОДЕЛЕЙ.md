# FAQ: Сравнение моделей эмбеддингов

## Общие вопросы

### Q: Какую модель выбрать?

**A:** Зависит от вашего железа:

- **GTX 1060 3GB + CUDA**: E5-Small с CUDA
- **Только CPU**: RuBERT-Tiny2
- **Нужно максимальное качество**: FRIDA (но медленно)
- **Real-time приложения**: RuBERT-Tiny2

### Q: Почему compare_models.py показывает "N/A*" для поиска?

**A:** База данных создана с другой моделью. Разные модели создают эмбеддинги разной размерности:
- FRIDA: 1536
- E5-Small: 384
- MiniLM-L12: 384
- RuBERT-Tiny2: 312

ChromaDB не может использовать эмбеддинги разной размерности в одной коллекции.

**Решение**: Используйте `test_single_model.py` для тестирования конкретной модели.

### Q: Как протестировать поиск с новой моделью?

**A:** 
```bash
# Вариант 1: Быстрый тест без пересоздания базы
python test_single_model.py intfloat/multilingual-e5-small cuda
# Покажет предупреждение о несовместимости базы

# Вариант 2: Полный тест с пересозданием базы
# 1. Обновите config.yaml
# 2. Пересоздайте базу
python load_tnved.py
# 3. Протестируйте
python test_single_model.py intfloat/multilingual-e5-small cuda
```

### Q: Нужно ли пересоздавать базу при смене модели?

**A:** Да, обязательно! Разные модели создают эмбеддинги разной размерности. Старая база не будет работать с новой моделью.

### Q: Сколько времени занимает пересоздание базы?

**A:** Зависит от модели и железа:
- FRIDA на CPU: ~30-60 минут для 6000 записей
- E5-Small на CUDA: ~10-15 минут
- RuBERT-Tiny2 на CPU: ~5-10 минут

### Q: Можно ли использовать несколько моделей одновременно?

**A:** Да, но нужно создать отдельные коллекции для каждой модели:

```yaml
# config_frida.yaml
database:
  collection_name: "tnved_frida"

# config_e5.yaml
database:
  collection_name: "tnved_e5"
```

Но это требует больше места на диске (каждая коллекция хранит свои эмбеддинги).

## Технические вопросы

### Q: Почему E5-Small быстрее FRIDA?

**A:** Несколько причин:
1. Меньший размер модели (470 MB vs 3.1 GB)
2. Меньше параметров для вычисления
3. Меньшая размерность эмбеддингов (384 vs 1536)
4. Оптимизированная архитектура

### Q: Потеряю ли я качество при переходе на E5-Small?

**A:** Качество будет немного ниже, но разница обычно незначительна для большинства задач. E5-Small специально оптимизирована для баланса скорости и качества.

Рекомендуем протестировать на ваших данных:
```bash
python test_single_model.py intfloat/multilingual-e5-small cuda
```

### Q: Что означает "similarity score"?

**A:** Это мера схожести между запросом и результатом (от 0 до 1):
- **0.8-1.0**: Очень похоже
- **0.6-0.8**: Похоже
- **0.4-0.6**: Средняя схожесть
- **0.0-0.4**: Слабая схожесть

### Q: Почему RuBERT-Tiny2 такая маленькая?

**A:** Это "distilled" (дистиллированная) модель - она обучена копировать поведение большой модели, но с меньшим количеством параметров. Это компромисс между размером и качеством.

## Проблемы и решения

### Q: Ошибка "Collection expecting embedding with dimension of X, got Y"

**A:** База данных создана с другой моделью. Решение:
```bash
# Обновите config.yaml с нужной моделью
python load_tnved.py
```

### Q: Ошибка "CUDA out of memory"

**A:** GPU не хватает памяти. Решения:
1. Уменьшите batch_size в config.yaml
2. Используйте более компактную модель (RuBERT-Tiny2)
3. Переключитесь на CPU: `device: "cpu"`

### Q: Модель загружается очень долго

**A:** Первый запуск загружает модель из HuggingFace. Последующие запуски будут быстрее (модель кэшируется локально в `~/.cache/huggingface/`).

### Q: compare_models.py зависает

**A:** Возможно, модель слишком большая для вашего железа. Попробуйте:
1. Закрыть другие приложения
2. Тестировать модели по одной: `test_single_model.py`
3. Использовать CPU вместо CUDA

## Производительность

### Q: Какая модель самая быстрая?

**A:** RuBERT-Tiny2 - примерно в 10 раз быстрее FRIDA.

### Q: Какая модель требует меньше всего памяти?

**A:** RuBERT-Tiny2 - всего 120 MB (в 25 раз меньше FRIDA).

### Q: Можно ли ускорить FRIDA?

**A:** Не сильно. FRIDA - большая модель, и её размер определяет скорость. Лучше перейти на более компактную модель.

### Q: Стоит ли использовать CUDA?

**A:** Да, если у вас есть GPU:
- E5-Small на CUDA: ~3-5x быстрее чем на CPU
- RuBERT-Tiny2 на CUDA: ~5-8x быстрее чем на CPU

Но для FRIDA на GTX 1060 3GB может не хватить памяти.

## Использование

### Q: Как часто нужно пересоздавать базу?

**A:** Только при:
1. Смене модели
2. Добавлении новых данных
3. Изменении нормализации текста

### Q: Можно ли использовать эти модели для других языков?

**A:** 
- **FRIDA**: Только русский
- **E5-Small**: Мультиязычная (100+ языков)
- **MiniLM-L12**: Мультиязычная (50+ языков)
- **RuBERT-Tiny2**: Только русский

### Q: Какая модель лучше для production?

**A:** E5-Small - хороший баланс скорости, качества и надежности. От Microsoft, активно поддерживается.
