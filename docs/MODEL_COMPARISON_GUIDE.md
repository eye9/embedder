# Руководство по сравнению моделей эмбеддингов

## Обзор

Этот документ описывает процесс сравнения различных моделей эмбеддингов для системы поиска по ТНВЭД.

## Сравниваемые модели

### 1. FRIDA (текущая модель)
- **ID модели**: `ai-forever/FRIDA`
- **Размер**: ~3.1 GB
- **Особенности**: Специализированная модель для русского языка от AI Forever
- **Плюсы**: Высокое качество для русского текста
- **Минусы**: Большой размер, медленная работа

### 2. E5-Small (рекомендуется)
- **ID модели**: `intfloat/multilingual-e5-small`
- **Размер**: ~470 MB
- **Особенности**: Компактная мультиязычная модель от Microsoft
- **Плюсы**: Быстрая, поддержка CUDA, хорошее качество
- **Минусы**: Чуть ниже качество для русского по сравнению с FRIDA

### 3. MiniLM-L12
- **ID модели**: `sentence-transformers/paraphrase-multilingual-MiniLM-L12-v2`
- **Размер**: ~470 MB
- **Особенности**: Популярная мультиязычная модель
- **Плюсы**: Надежная, проверенная временем
- **Минусы**: Средняя скорость

### 4. RuBERT-Tiny2
- **ID модели**: `cointegrated/rubert-tiny2`
- **Размер**: ~120 MB
- **Особенности**: Очень компактная модель для русского языка
- **Плюсы**: Самая быстрая, минимальные требования к памяти
- **Минусы**: Может быть ниже качество на сложных запросах

## Запуск сравнения

### Установка зависимостей

```bash
pip install tabulate
```

### Запуск скрипта

```bash
python compare_models.py
```

### Использование CUDA (если доступна)

Отредактируйте `config.yaml`:

```yaml
model:
  device: "cuda"  # Вместо "cpu"
```

## Метрики сравнения

Скрипт измеряет следующие метрики:

1. **Время загрузки модели** - сколько времени требуется для инициализации
2. **Размерность эмбеддингов** - размер векторного представления
3. **Скорость генерации** - количество эмбеддингов в секунду
4. **Скорость поиска** - среднее время выполнения поискового запроса
5. **Качество поиска** - средний score схожести для тестовых запросов

## Тестовые запросы

Скрипт использует 10 типичных запросов для оценки качества:

- Кофе в зернах арабика
- Свежие яблоки

- Натуральный мед

## Интерпретация результатов

### Скорость vs Качество

- **Для продакшена с высокой нагрузкой**: E5-Small или RuBERT-Tiny2
- **Для максимального качества**: FRIDA
- **Для баланса**: MiniLM-L12

### Требования к памяти

- **GPU < 2GB**: RuBERT-Tiny2 (CPU режим)
- **GPU 2-4GB**: E5-Small или MiniLM-L12
- **GPU > 4GB**: Любая модель, включая FRIDA

## Переключение модели

После выбора модели обновите `config.yaml`:

```yaml
model:
  name: "intfloat/multilingual-e5-small"  # Ваша выбранная модель
  device: "cuda"  # или "cpu"
```

Затем пересоздайте базу данных эмбеддингов:

```bash
python load_tnved.py
```

**Важно**: При смене модели необходимо пересоздать базу данных, так как размерность эмбеддингов может отличаться.

## Дополнительные рекомендации

### Для GTX 1060 3GB

- Используйте E5-Small или RuBERT-Tiny2 с CUDA
- Установите batch_size: 16-32 в config.yaml
- Мониторьте использование памяти GPU

### Для CPU-only систем

- RuBERT-Tiny2 - лучший выбор для скорости
- E5-Small - хороший баланс
- Увеличьте batch_size до 100-200

### Оптимизация производительности

1. Используйте CUDA если доступна
2. Настройте batch_size под вашу систему
3. Рассмотрите кэширование частых запросов
4. Используйте более компактную модель для real-time приложений
