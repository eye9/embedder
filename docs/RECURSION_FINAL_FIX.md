# Финальное исправление ошибки рекурсии

## Проблема
Несмотря на исправления циклических импортов, ошибка "maximum recursion depth exceeded" все еще возникает при обработке файлов через веб-интерфейс.

## Анализ
1. ✅ Прямая обработка файлов работает без ошибок
2. ✅ TNVED интеграция инициализируется без ошибок  
3. ❌ Ошибка возникает только при обработке через веб-API

## Решение
Временно отключить TNVED интеграцию и использовать только fallback обработку до полного решения проблем с окружением Python.

## Реализация
1. Модифицировать `process_file_sync` для принудительного использования fallback
2. Добавить конфигурационный флаг для отключения TNVED интеграции
3. Обеспечить корректную работу без TNVED кодов
