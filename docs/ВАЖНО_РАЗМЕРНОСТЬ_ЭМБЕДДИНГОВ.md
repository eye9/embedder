# ⚠️ Важно: Размерность эмбеддингов

## Проблема

Каждая модель создает эмбеддинги **разной размерности**:

| Модель | Размерность |
|--------|-------------|
| FRIDA | 1536 |
| E5-Small | 384 |
| MiniLM-L12 | 384 |
| RuBERT-Tiny2 | 312 |

## Почему это важно?

ChromaDB хранит эмбеддинги определенной размерности в коллекции. Если вы создали базу данных с FRIDA (размерность 1536), вы **не можете** использовать её с E5-Small (размерность 384).

## Ошибка

```
chromadb.errors.InvalidArgumentError: Collection expecting embedding with dimension of 1536, got 384
```

Эта ошибка означает, что база данных создана с одной моделью, а вы пытаетесь использовать другую.

## Решение

### Вариант 1: Пересоздать базу данных (рекомендуется)

```bash
# 1. Обновите config.yaml с новой моделью
# model:
#   name: "intfloat/multilingual-e5-small"

# 2. Пересоздайте базу данных
python load_tnved.py

# 3. Теперь можно использовать поиск
python search_tnved.py
```

### Вариант 2: Использовать разные коллекции

Можно создать отдельные коллекции для каждой модели:

```yaml
# config.yaml для FRIDA
database:
  collection_name: "tnved_frida"

# config.yaml для E5-Small
database:
  collection_name: "tnved_e5small"
```

Но это требует больше места на диске.

## Как работают скрипты сравнения

### `test_single_model.py`
✅ **Работает всегда**
- Загружает модель
- Тестирует генерацию эмбеддингов
- Пытается выполнить поиск
- Если база несовместима, показывает предупреждение

### `compare_models.py`
⚠️ **Частично работает**
- Загружает все модели
- Тестирует генерацию эмбеддингов для всех
- Тестирует поиск **только для модели, с которой создана база**
- Для остальных моделей показывает "N/A*"

### `compare_search_results.py`
❌ **Требует пересоздания базы**
- Требует, чтобы база была создана с каждой моделью
- Иначе будет ошибка размерности

## Рекомендуемый workflow

### Для быстрого тестирования

```bash
# 1. Протестируйте модель без поиска (только скорость загрузки/генерации)
python compare_models.py

# 2. Выберите понравившуюся модель

# 3. Протестируйте её с поиском
python test_single_model.py intfloat/multilingual-e5-small cuda

# 4. Если база несовместима, пересоздайте её
# Обновите config.yaml
python load_tnved.py

# 5. Повторите тест
python test_single_model.py intfloat/multilingual-e5-small cuda
```

### Для детального сравнения поиска

Если вы хотите сравнить качество поиска разных моделей:

```bash
# 1. Тест FRIDA
# config.yaml: name: "ai-forever/FRIDA"
python load_tnved.py
python test_single_model.py ai-forever/FRIDA

# 2. Тест E5-Small
# config.yaml: name: "intfloat/multilingual-e5-small"
python load_tnved.py
python test_single_model.py intfloat/multilingual-e5-small cuda

# 3. Тест RuBERT-Tiny2
# config.yaml: name: "cointegrated/rubert-tiny2"
python load_tnved.py
python test_single_model.py cointegrated/rubert-tiny2 cuda
```

Да, это требует времени, но это единственный способ корректно сравнить качество поиска.

## Почему нельзя просто конвертировать?

Эмбеддинги разных моделей находятся в **разных векторных пространствах**. Нельзя просто "дополнить нулями" или "обрезать" вектор - это полностью изменит семантику.

Каждая модель обучена по-своему и создает свое уникальное представление текста.

## Итого

✅ **Можно делать без пересоздания базы:**
- Тестировать скорость загрузки модели
- Тестировать скорость генерации эмбеддингов
- Сравнивать размер моделей

❌ **Требует пересоздания базы:**
- Тестировать поиск
- Сравнивать качество результатов
- Сравнивать similarity scores

**Рекомендация**: Используйте `test_single_model.py` для быстрого тестирования одной модели. Это самый простой способ оценить модель перед полным переходом.
